# Evidence Ledger

This ledger tracks whether tested results increase, decrease, or merely constrain confidence in the Origin Reframe.

The target is not confirmation. The target is evidence quality.

## Evidence Scale

| Rating | Meaning |
| --- | --- |
| `SUPPORTIVE` | Result survives controls and is naturally anticipated by the Origin Frame. |
| `CONSTRAINING` | Result fits part of the frame but also reveals limits or non-specificity. |
| `NEUTRAL` | Result is reproducible but does not distinguish the Origin Frame from ordinary explanations. |
| `CHALLENGING` | Result weighs against a proposed Origin-frame hypothesis. |
| `UNTESTED` | Not yet locally checked. |

## What Would Increase Weight

- Origin metrics improve prediction after ordinary baselines are included.
- A metric defined on the Origin shell map predicts behavior before any conjecture-specific tuning.
- A pre-registered Origin metric identifies where a pattern should strengthen, weaken, or fail.
- The same metric transfers across distinct phenomena: factorization, prime gaps, Gilbreath rows, and Goldbach witnesses.
- Controls fail where the Origin metric predicts failure, not merely where a post-hoc interpretation can be supplied.

## What Would Decrease Weight

- Conventional baselines explain all observed structure.
- Origin metrics add no predictive or discriminating value.
- The frame only re-describes known facts after the fact.
- Similar behavior appears in controls where the frame predicts it should not.

## Current Bottom Line

The Origin Reframe is operationalized but not established. Current local evidence is mixed, and the repo has been over-weighted toward conjecture probes because they are easier to test than the broader Emanation claim. The Origin structure scan now restores the proper center: shells, return paths, branching, and compression are the native objects; Gilbreath, Goldbach, twin primes, and prime gaps are downstream transfer tests. The direct size-banded Origin controls are supportive for the measurement posture: shell depth retains strong residual structure after the number-line size baseline is removed. The modular-return scans add the clearest Origin-first transfer line so far. E21 narrows the claim to `lambda/phi` return-exponent compression; E22 shows within-shell branching/compression signal; E23 shows that narrowed within-shell signal is stable across tested ranges; E24 gives a theorem-level mechanism via Carmichael lcm overlap; E25 prevents overclaiming by ruling out radical-compression monotonicity; and E26 shows that the exact shell-conditioned log signal is overwhelmingly an overlap-pressure signal, with Origin-facing proxies capturing only part of it. E27 is an important brake: the same overlap-pressure mechanism did not clearly predict held-out large prime gaps. E28 adds the first Pakheta-informed anchor test: fixed-point richness is exact and well controlled, but fully explained by CRT component branching. E29 closes the immediate transfer question for those exact anchor definitions: once their component formulas are removed, no leftover anchor variance remains to transfer. E30 opens the next phase: an actual Origin-Pakheta path calculus with exact commutators and finite controls. E31 closes the loop on the v0 neighborhood transfer: once the classical Carmichael kernel overlap and the 2-adic lambda defect are conditioned out, the strong `C/N_-` modular transfer drops below the repo's transfer bar, so the preprint Section 5.1 result should be read as mechanism re-measurement rather than novel path-order transfer. E32 completes that closure at scale: the marginal post-knockout trace proved real but classical, fully absorbed by a third named channel (power-kernel overlap), so the entire `C/N_-` modular coupling now decomposes into three classical covariates and the three-covariate ladder is the minimum baseline for future v0 transfer claims. E33 dissolves the preprint Section 5.2 prime-gap claim at scale: the marginal enrichment did not replicate in larger, later windows, leaving the v0 boundary operators with no demonstrated prime-gap reach. The honest state of the v0 calculus is therefore: exact and verified as algebra, with no surviving transfer result; the open directions are the systematic commutator catalog and pre-registered predictions on targets independent of the factorization generator. E34 opens the catalog phase: all `105` operator pairs are now classified, twelve catalog lemmas are proven (five read directly off the instrument's first candidate queue), and the open queue is a ranked list of `65` loci with the `phi(sigma) = sigma(phi)` coincidence set at its head. The Goldbach witness test weighs against treating simple factor-depth metrics as distinctive Origin evidence, because the conventional singular-factor baseline absorbs most of that signal. Gilbreath remains a useful boundary-return probe, but k-history Markov controls show that local gap memory is a serious null model. The prime-gap origin-profile scan is candidate-supportive at the residual-correlation level, but held-out prediction is constraining.
